# plae — DESIGN.md

plae 웹앱의 디자인 시스템. 새 화면을 만들거나 기존 화면을 손볼 때 이 문서를 기준으로 한다.
(awesome-design-md 방식의 단일 문서 — LLM·사람 모두 이걸 보고 UI를 맞춘다.)

## 컨셉 — "악기 패널 (Instrument Panel)"

plae는 화면이 **한 대의 음악 장비**처럼 보인다. 드럼 머신·모듈러 신스·믹싱 콘솔의
촉각적 질감 — 따뜻한 종이(데스크) 위에 어두운 하드웨어 모듈이 놓인 모습.
중1 대상이라 친근하지만, 장난감이 아니라 "진짜 악기"의 인상을 준다.

원칙:
- 컨테이너는 "모듈" — 어두운 패널 카드가 크림 배경 위에 떠 있다.
- 라벨은 실크스크린 — 모노스페이스 대문자, 자간 넓게, 작게, 앞에 ■ 틱.
- 버튼·패드는 물리적 — 누르면 아래로 눌리고 그림자가 압축된다.
- 색은 절제하되 액센트는 강하게 — 버밀리언 하나가 화면을 끈다.

## 타이포

- **Space Mono** (`font-mono`, 400/700) — 워드마크, 라벨, 숫자, 영문.
  라벨은 `uppercase` + 넓은 `tracking`. 하드웨어 실크스크린 느낌.
- **IBM Plex Sans KR** (`font-sans`, 400~700) — 모든 한국어 UI·본문.

본문 15~16px / 라벨 11~12px / 제목은 Space Mono로 크게.

## 컴포넌트 클래스 (`index.css` `@layer components`)

마크업에서 base + variant를 조합한다: `class="btn btn-accent"`.

- `.panel` — 어두운 모듈 카드 (둥근 모서리 + 아래쪽 물리 그림자).
- `.panel-label` — 패널/섹션 상단 실크스크린 라벨.
- `.btn` + `.btn-accent` / `.btn-dark` / `.btn-ghost` — 촉각 버튼 (누르면 눌림).
- `.pad` — 드럼 패드·큰 터치 타깃 (두꺼운 아래 그림자).
- `.field` (패널 위) / `.field-light` (paper 위) — 입력칸.
- `.chip` — 작은 모노 태그.
- `.animate-rise` — 아래에서 떠오르는 등장 (로드 시 `animation-delay`로 stagger).

## 모션

- 버튼·패드: `:active`에서 `translateY` + 그림자 압축 — 물리적 눌림.
- 페이지 로드: 패널이 `.animate-rise`로 떠오르며 stagger.
- 재생/활성: `signal` 색.
- 과하지 않게 — 잘 짜인 로드 한 번이 산만한 마이크로 인터랙션보다 낫다.

## 레이아웃

- 태블릿 우선. 터치 타깃 최소 44px.
- 페이지: paper 배경(미세 그레인) + 상단 어두운 콘솔 헤더 + 중앙 정렬 본문 컬럼.
- 패널은 여백을 두고 떨어뜨려 배치 — "데스크 위 장비들".
