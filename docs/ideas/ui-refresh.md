# UI 리프레시 — 아이디어 스케치

> 상태: **아이디어 단계**. 아직 착수 안 함. Phase 5(수업 직전 검증)와 별개 트랙.

## 동기

현재 프론트엔드(`apps/web`)는 Tailwind 기본 팔레트(slate/indigo) 위주의 기능 우선 UI다.
동작에는 문제없지만, 사용자가 **중1 학생**인 점을 고려하면 더 또렷한 톤·재미·동기 부여가 있는
화면이 수업 몰입에 도움이 된다. 객관 모드의 레이어 작곡기처럼 "쌓는" 인터랙션은
시각적으로 더 살릴 여지가 크다.

## 참고 리소스

**awesome-design-md** — https://github.com/VoltAgent/awesome-design-md

인기 사이트들의 디자인 시스템을 `DESIGN.md` 한 장으로 정리해둔 모음. 색 팔레트·타이포·
컴포넌트 스타일·여백 규칙·디자인 철학이 LLM이 파싱하기 좋은 마크다운으로 적혀 있어,
원하는 파일을 프로젝트에 넣고 에이전트에게 "이 느낌으로 만들어줘"라고 지시하는 용도.
73개+ 파일, AI 플랫폼·개발툴·SaaS·미디어 등 10개+ 카테고리.

→ 활용 방향: 후보 `DESIGN.md` 몇 개를 골라 plae 톤에 맞게 발췌·혼합한
**plae 전용 `DESIGN.md`**를 만들고, 그걸 기준으로 컴포넌트를 다시 입힌다.

## 작업 시 고려할 것 (착수 전 정리용)

- **대상**: 중1. 너무 기업용(Linear/Notion식 미니멀)으로 가면 차갑다. 친근하되 유치하지 않게.
- **범위**: 토큰(색·타이포·radius·spacing) → 공통 컴포넌트(버튼·입력·카드) → 화면별 순.
  `LoginScreen` / `HomeScreen` / `ComposeScreen` / `StrudelComposer` / `TeacherDashboard`.
- **레이어 작곡기**: 레이어 stack을 시각적 블록으로 보여주면 "음악을 쌓는다"는 개념이 직관적.
- **제약 유지**: 일회성 수업 앱 — 디자인 시스템 라이브러리 도입 같은 과한 투자는 지양.
  Tailwind 설정 + 기존 컴포넌트 리스타일 수준에서.
- **태블릿 우선**: 학생은 태블릿. 터치 타깃 크기·가독성을 데스크탑보다 우선.
